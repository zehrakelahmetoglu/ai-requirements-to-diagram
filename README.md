# ai-requirements-to-diagram


Günümüzde Epic ve User Story’ler genellikle manuel olarak analiz edilip diyagramlara dönüştürülüyor. Bu süreç hem zaman alıcı hem de hataya açık bir süreçtir. Özellikle büyük projelerde bu durum verimlilik kaybına neden olmaktadır.

Problemi çözmek için yapay zekâ destekli bir sistem geliştirilmesi amaçlanmaktadır. Sistem, kullanıcıdan alınan Epic ve User Story’leri analiz ederek metin içerisinden aktörleri ve işlemleri otomatik olarak çıkarmaktadır.

Elde edilen bu bilgiler doğrultusunda sistem, otomatik olarak yazılım diyagramları üretmektedir. Bu diyagramlar arasında Use Case, Activity, Sequence ve Class diyagramları bulunmaktadır.

Projenin genel akışı şu şekildedir: Kullanıcı metni girer, bu metin API aracılığıyla backend’e iletilir, backend ise yapay zekâ modülüne gönderir. Yapay zekâ metni analiz eder ve elde edilen sonuçlar tekrar sisteme iletilerek diyagram üretimi gerçekleştirilir.


Sonuç olarak bu proje ile gereksinimlerden sistem modeline geçiş sürecini otomatikleştirerek hem zaman kazancı sağlanmasıhem de hataların azaltılması hedeflenmektedir.
